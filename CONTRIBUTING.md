
  ```

  ```

  ```bash
  git push origin feature/your-feature-name
  ```

### 2. Reporting Issues

* Use the **Issues tab** to report bugs, request features, or ask questions.
* Include **clear steps to reproduce** and screenshots if possible.
* Label your issues appropriately (bug, enhancement, question).

### 3. Code Style

* Follow the **existing code style** in the repository.
* Use meaningful variable and function names.
* Keep functions and modules **small and focused**.

### 4. Review Process

* Pull Requests will be **reviewed by maintainers** before merging.
* Address comments and make changes requested during the review.
* Ensure all **tests pass** before requesting a merge.

### 5. Thank You!

By contributing, you help improve the project, enhance community collaboration, and share your knowledge.
